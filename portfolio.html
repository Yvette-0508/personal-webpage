Core Components

PIT-Correct Data Kernel & Feature Library

Built a bi-temporal fact store (Parquet + DuckDB + Polars) where every read requires an explicit as_of timestamp, enforced by custom Ruff rules and Pydantic strict types. On top, assembled PIT-safe tradable universes and a five-factor library (value, momentum, quality, low-vol, size), each shipped with IC time series, decile spreads, and bootstrap p-values. Point-in-Time Polars/DuckDB Factor Library XGBoost Fusion

Portfolio Construction & Six-Part Validation Engine

Signal-to-weights layer with three optimizers (MVO with Ledoit-Wolf shrinkage, Risk Parity, HRP) under position, sector, turnover, and leverage constraints. Built on vectorbt, every strategy must pass a six-part validation gate — walk-forward Purged K-Fold, bootstrap Sharpe distribution, subsample stability, turnover-adjusted Sharpe, Monte Carlo signal scramble, and factor attribution — before promotion. In-sample Sharpe alone never qualifies.

cvxpy/HRP Walk-Forward Bootstrap Sharpe Validation Gate

Agent-Orchestrated Execution & Drift Monitoring

Unified broker abstraction (Alpaca / IBKR / Binance Testnet) converting target weights into fills, with live-vs-backtest drift detection across returns, turnover, and factor exposures. The LLM layer (Zen CLI, LiteLLM) acts as orchestrator — not oracle — producing declarative StrategySpec objects and sandboxed Python (Docker, no network, read-only fact copy). Agents cannot place orders, mutate the kernel, or bypass validation.
